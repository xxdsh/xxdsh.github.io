JAVA实战学习笔记

November 10, 2021 6098

跨平台/可移植性 安全性 面向对象 简单性 高性能 分布式 多线程 健壮性
Java = 编译型(编译器编译成字节码) + 解释型(虚拟机JVM(Java Virtual Machine)(解释器)解释执行 屏蔽底层运行平台差异)
Jave Runtime Environment(JRE):Java虚拟机 库函数 运行Java应用程序所必须的文件
Java Development Kit(JDK):包含JRE,以及增加编译器和调试器等用于程序开发的文件
JavaSE(Java Standard Edition):标准版，个人计算机上的应用；JavaEE(Java Enterprise Edition):企业版，定位在服务器端的应用； JavaME(Java Micro Edition):微型版，定位在消费性电子产品的应用上
局部变量 成员变量 静态变量的区别
声明常量：final type varName = value 字符常量 符号常量
基本数据类型(数值型(byte short int long float double) 字符型(char) 布尔型(boolean)) + 引用数据类型(类(class) 接口(interface) 数组) BigDecimal和BigInteger 注意浮点数比较(精度影响)
BigDecimal bd = BigDecimal.valueOf(1.0);
bd = bd.subtract(BigDecimal.valueOf(0.1));
char 2个字节 unicode 65535 \u
算术运算符 + - * / % 注意运算结果的类型 余数符号和左边操作数相同
逻辑与& 短路与&& 位运算
面向过程和面向对象 软件分析、设计和开发 面向对象分析(OOA) 面向对象设计(OOD) 面向对象编程(OOP) 面向过程是一种“执行者思维” 面向对象是一种“设计者思维” 面向对象离不开面向过程
程序技术器(Program Counter Register 每个线程有自己的程序计数器 存储当前线程正在执行的Java方法的JVM指令地址) 虚拟机栈(Java Virtal Machine Statc 线程私有 栈帧) 本地方法栈(Nativate Method Statck 调用本地方法时使用的栈) 方法区(又叫静态区 所有线程共享，存储被虚拟机加载的元数据，包括类信息、常量、静态变量、即时编译器编译后的代码) 运行时常量池(Run-time Constant Pool 方法区中 存放字面量和符号引用) 堆(几乎所有创建的Java对象实例都是直接分配到堆上的) 新生代 老年代 直接内存(属于Java虚拟机运行时数据区的一部分) 其它内存
虚拟机的内存可以分为三个区域：虚拟机栈stack 堆heap 方法区method area
同一类的对象有不同的成员变量存储空间； 同一类的每个对象共享该类的方法
对中对象的管理 对象空间的分配 对象空间的释放 垃圾回收过程：发现无用的对象；回收无用对象占用的内存空间； 垃圾回收相关算法：引用计数法；引用可达法；
分代垃圾回收机制：不同的对象的生命周期是不一样的； 年轻代(Minor GC 效率较高 操作频繁 浪费内存空间) 年老代(Major GC和Full GC) 持久代(JDK8以后没有，使用metaspace元数据空间和堆替代) Eden Survivor(循环存取 存放垃圾回收后，仍然有用的对象) Tenured/Old 创建大量无用对象 静态集合类的使用(HashMap, Vector, List等) 各种连接对象未关闭(IO流对象 数据库连接对象 网络连接对象) 监听器的使用
有如下原因可能导致Full GC: 1. 年老代被写满 2.持久代被写满 3.System.gc()被显式调用 4.上一次GC之后Heap的各域分配策略动态变化
this的本质是“创建好的对象的地址” 创建对象的过程：分配空间，赋默认值，调用构造方法(this是隐藏参数)，返回对象的地址给相关的变量。 普通方法中this指向调用该方法的对象，构造方法中，this指向正要初始化的对象；使用this关键字调用重载的方法；this不能用于static方法
static方法不能调非静态方法
package 域名倒着写 java.lang.Object
静态导入(static import)：用于导入指定类的静态属性和静态方法
继承(代码复用，更加容易实现类的扩展；方便建模) 封装 多态(同一个方法调用，由于对象不同可能会有不同的行为 条件：继承 方法重写 父类引用指向子类对象) 抽象类 接口 内部类
对象 instanceof 类 (对当象是右面类或自雷所创建对象时，返回true)
重写Override 方法名、形参列表相同 + 返回值类型和声明异常类型，子类小于等于父类 + 访问权限，子类大于等于父类
final关键字的作用：修饰变量(被修饰的变量不可改变)； 修饰方法(该方法不可被子类重写)； 修饰类(修饰的类不能被继承)
组合的核心是将父类对象作为子类的属性，然后子类通过调用这个属性来获得父类的属性和方法
Object类时所有Java类的根基类，也意味着所有的Java对象都用于Object类的属性
toString “类名 + @ + 16进制的hashcode” 在打印输出或者用字符串连接对象时，会自动调用该对象的toString()方法
==：基本类型值相等，引用类型地址相等；equals对象内容相等 默认比较两个对象的hashcode
super可以看做是直接父类对象的引用，可以通过super来访问父类中被子类覆盖的方法或属性
构造方法的第一行代码没有显示的调用super()或者this()，那么Java默认都会调用super()，含义是调用父类的无参数构造方法
属性/方法查找顺序； 构造方法调用顺序；
封装：高内聚(类的内部数据操作细节自己完成，不允许外部干涉)，低耦合(仅暴露少量的方法给外部使用，经历方便外部调用) 安全性 复用性
访问控制符：private,default,protocted,public 同一个类 同一个包中 子类(可位于其他包) 所有类
若父类和子类在同一个包中，子类可以访问父类的protocted成员，也可访问父类对象的protocted成员
若父类和子类不在同一个包中，子类可以访问父类的protocted成员，不能访问父类对象的protocted成员
多态用法一：父类引用作为方法的形参，实参可以是任意的子类对象，可以通过不同的子类对象实现不同的行为方式
抽象方法：使用abstract修饰的方法，没有方法体，只有声明；包含抽象方法的类就是抽象类，要求子类必须实现具体实现。
接口是比抽象类还抽象的抽象类，可以更加规范的对子类进行约束。规范和具体实现的分离
接口定义了可以向外部提供的服务；接口定义了实现者能提供哪些服务
接口：访问修饰符(public或默认) 接口名 extentds 常量(public static final) 方法(public abstract) implements 不能创建实例，但是可用于声明引用变量类型 必须实现接口中所有方法
接口中定义的静态方法(从属于接口，可以直接通过接口名调用)和默认方法(使用default关键词) 默认方法中可以调用静态方法
Java字符串是Unicode字符序列 全局字符串常量池(每个VM中只有一份) class文件常量池(每个clss都有) 运行时常量池(将class常量池中的符号殷涌智转存到运行时常量池中) charAt equals equalsIgnoreCase indexOf lastIndexOf length replace startsWith endsWith substring toLowerCase toUpperCase trim
内部类：成员内部类(非静态内部类 + 静态内部类) + 匿名内部类 + 局部内部类； 内部类可以使用public default protected private以及static修饰； 内部类是相对独立的一种存在，其成员变量/方法名可以和外部类的相同 内部类可以直接访问外部类的私有属性， 内部类被当成其外部类的成员。 但外部类 不能访问内部类的内部属性 非静态内部类不能有静态方法、静态属性和静态初始化块 内部类里方法的局部变量：变量名。 内部类属性：this.变量名。 外部类属性：外部类名.this.变量名。 Outer1.Inner1 inner = new Outer1().new Inner1(); 静态内部类可以访问外部类的静态成员，不能访问外部类的普通成员。 静态内部类看做外部类的一个静态成员。

异常指程序运行过程中出现的非正常现象，例如用户输入错误、除数为零、需要处理的 文件不存在、数组下标越界等。
所谓异常处理，就是指程序在出现问题时依然可以正确的执行完。
Java 是采用面向对象的方式来处理异常的。
处理过程： 抛出异常：在执行一个方法时，如果发生异常，则这个方法生成代表该异常的一个对象，停止当前执行路径，并把异常对象提交给JRE。 
捕获异常：JRE得到该异常后，寻找相应的代码来处理该异常。JRE在方法的调用栈中查找，从生成异常的方法开始回溯，直到找到相应的异常处理代码为止。
java.lang.Throwable下面派生了两个子类：Error(程序无法处理的错误) 和 Exception(程序本身能够处理的异常)(空指针异常(NullPointerException)、 数组下标越界异常(ArrayIndexOutOfBoundsException)、 类型转换异常(ClassCastException)、算术异常(ArithmeticException)等)(RuntimeException 运行时异常 CheckedException 已检查异常)
