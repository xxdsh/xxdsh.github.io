自动化机器学习

Automated Machine Learning
自动化机器学习
可以通过反复迭代去搜寻针对特定业务场景的最优的数据预处理操作、超参数配置以及算法模型

自动化机器学习：针对特定的一类或若干类机器学习任务，在没有人类专家干预且计算资源有限的条件下，自动化地构造机器学习算法流程。
根据数据建立算法模型、算法效果评估、不断优化算法效果
主要目标：

  • 能够在不同数据集甚至不同任务之间泛化
  • 不需要人类专家干预
  • 计算效率(在有限的计算资源下，有限的时间内给出最优的算法流程

研究方向：

  • 自动化特征提取
  • 自动化模型选择
  • 自动化模型参数调优(技术：简单/启发式搜索、无梯度优化、强化学习、梯度下降优化)
  • 自动化模型结构搜索
  • 自动化模型评估
  • 元学习
  • 迁移学习

模型和超参数调优

机器学习模型的选择和超参数的指定统称为机器学习模型的配置。

  • 网格搜索
    • 需要考虑的三个问题：
      • 配置参数空间通常是一个有层次的空间，即有一些参数的存在是依赖于另一些参数的取值的。因此，在对配置参数空间进行划分网格时，需要分层次划分。
      • 配置参数的类型
      • 配置参数的取值范围
  • 随机搜索
  • 贝叶斯优化
    • 先随机尝试一些配置参数，并训练验证得到效果指标
    • 根据这些的采样值。通过贝叶斯公式推断出在任意下的后验概率分布
    • 根据这个后验分布可以选择一个在当前已知信息下最优的作为下一次训练验证尝试的配置参数

建模方法：高斯过程回归； 随机森林； 树形Parzen估计； 深度神经网络；
获得函数(Acquisition Function): 期望提升(Exoected Improvement)；上限置信界(Upper Confidence Bound)； 知识梯度(Knowledge Gradient)

神经网络架构搜索

一次架构搜索(One-shot Architecture Search)
可微架构搜索(Differentiable Architecture Search, DARTS)
搜索空间 搜索策略(选定网络架构) 评估策略(评估的性能)

  • 搜索空间
    基本的链式架构->多分支、更复杂的架构
  • 搜索策略
    • 与神经网络上的超参数自动化调优是等同的
    • 演化算法
      • 从架构群中选择架构的策略，从亲代架构生成子代架构的流程、架构的淘汰标准
    • 强化学习算法
    • 基于梯度的优化算法
  • 评估策略

一次架构搜索的基本步骤：
(1)设计一个能覆盖所有候选架构的超级架构
(2)训练该超级架构，使之能够用来预测子架构在验证集上的性能
(3)每选出一个子架构，就用预训练过的超级架构对其在验证集上的性能进行评估
(4)从所有候选子架构中选出效果最好的一个，重新训练并在测试集上得到最终的性能指标
