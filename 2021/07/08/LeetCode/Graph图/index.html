最短路径算法

并查集 Disjoint-set/Union-find forest

find(x): find the root/cluster-id of x

union(x, y): merge two clusters

check whether two elements are in the same set or not in O(1)

Find: O(α(n))* ≈ O(1)

Union: O(α(n))* ≈ O(1)

Space: O(n)

without optimization: Find: O(n)

Two key optimizations: 1)path compression: make tree flat(happened during find ); 2) union by rank: merge low rank tree to high rank tree

* : amortized α(.) : inverse Ackermann function

Time Complexity: O(mn * max(m, n))

Space Complexity: O(mn)

方法二：Binary Search + BFS

If we have a budget, we can reduce this problem to whether there is a path from (0,0) - (n - 1, m - 1) that the max edge weight is within the budget.

this can be solved in O(mn) using BFS

Then we can use binary search to guess the budget and find the lowest one that has a path

Time Complexity: O(mn*log(max(h)-min(h)))

方法三：Dijkstra

dist[v] = min dist from(0, 0) to v

dist[v] = dist[u] + cost(u, v)

dist[v] = max(dist[u] + cost(u, v))

Time Complexity: O(Elogv) = O(mnlogmn)

964. Least Operators to Express Number(表示数字的最少运算符)

some observations:

  1. ops(-k) = ops(+k) -4 = -3/3-3 +4 = +3/3+3
  2. Total ops = # of x - 1 8 = - 3/3 + 3 * 3 = 3*3 - 3/3
  3. Any number can be expressed as T = c_0 * x^0 + c_1 * x^1 + …… + c_n * x^n - 1
    • ops = |c_0| * 2 + |c_1| + |c_2| * 2 + …… + |c_n| * n
  4. 1 can be only expressed by x/x
  5. There is no need to have term x^(k + 2) where k = [logx(target)]
  6. opt solution can start with T > target
  7. Division is only helpful to generate 1
  8. If t < x f(t, x) = min{2 * t - 1, (# x/x + x/x/ + x/x), 2 * (x - t), # x - x/x + x/x}

方法一：Dijkstra

find the shortest path from target to 0

方法二：DFS + Memorization

956. Regions Cut By Slashes (由斜杠划分区域)

方法一：Union Find(Faces)

Divide each grid into 4 triangles, and merge those triangles accordingly

ans = # of roots in the DSU

inner connection： case’ ‘:merge(0,1), merge(1,2), merge(2,3); case ‘/‘:merge(0,3), merge(1,2) case ‘\':merge(0,1), merge(2,3)

inter connection : 考虑右边和下边 merge(index + 1, (index + 4) + 3); merge(index + 2, (index + 4 * n) + 0)

方法二：Counting

Euler’s formula : V - E + F = C + 1 ——-> F = E - V + C + 1

ans = F - 1 = E - V + C

F : faces E: edges V: vertex C: components

It will be a little tricky to count E/V/C

We only care about when a new face is created. A new face will be created if the two endpoints of a new edge are already in the same connected components.

399. Evaluate Division(除法求值)

方法一：Graph + DFS

Time complexity: O(e + q * e)

方法二：Union Find

A/B = 2 -> parents[A] = {B, 2} parents[B] = {B, 1.0}

B/C = 3 -> parents[C] = {B, 1.0/3}

A / C ? p[A].key == p[C].key == B = p[A].val / p[C].val = 2 / (1.0 / 3) = 6

Time complexity: O(e + q)

字典 wordList 中从单词 beginWord 和 endWord 的 转换序列 是一个按下述规格形成的序列：

  • 序列中第一个单词是 beginWord 。
  • 序列中最后一个单词是 endWord 。
  • 每次转换只能改变一个字母。
  • 转换过程中的中间单词必须是字典 wordList 中的单词。
