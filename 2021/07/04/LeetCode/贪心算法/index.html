数据结构：Fenwick Tree/Binary Indexed Tree

Fenwick Tree was proposed to solve the prefix sum problem
The idea is to store partial sum in each node and get total sum by traversing the tree from leaf to root. The tree has a height of log(n)
Query:O(logn)
Update:O(logn)
lowbit(x) = x & (-x) = x & (~x+1)
Update Tree: 父节点和子节点间的关系：p = c + lowbit(c)
Query Tree: 父节点和子节点间的关系：p = c - lowbit(c)

321. Create Maximum Number(拼接最大数)

方法一：Greedy + DP

To solve the original problem, we need to solve two subproblems first

  1. maxNumber(nums, k)
    Pick element, keep popping back if the new one is greater than the back. Pick the first k as ans.
  2. maxNumber(nums1, nums2)
    Pick the front one from the larger array
    maxNumber(nums1, nums2, k) = max(maxNumber(maxNumber(nums1,i),maxNumber(nums2,k-i)))
    从nums1中取i个数，从nums2中取k-i个数，0 <= i <= min(k, n1) k-i<=n2 -> i >= k - n2

1520. Maximum Number of Non-Overlapping Substrings(最多的不重叠子字符串)

方法一：Naive Brute Force

Each character has three choices:(1)Start (2)Follow (3)End
Time complexity: O(n^3)
Space complexity: O(n)

方法二：Brute Force

A valid substring must contain all accurrences of any character it includes. We only need to make decision at the first accurance of each letter. At most 26 starting points.

方法三：Greedy

Observation: If a valid substring contains shorter valid strings, ignore the longer one and use the shorter one.
e.g. “abbeefba” is a valid substring, however, it includes “bbeefb”, “ee”, “f” three valid substrings, thus it won’t be part of the optimal solution, since we can always choose a shorter one, with potential to have one or more non-overlapping substrings. For “bbeefb, again it includes “ee” and “f”, so it won’t be optimal either. Thus, the optimal ones are “ee” and “f”.

  1. We just need to record the first and last occurrence of each character
  2. When we meet a character for the first time we must include everything from current pos to it’s last position. e.g. “abbeefba” | ccc, from first ‘a’ to last ‘a’, we need to cover “abbeefba”
  3. If any character in that range has larger end position, we must extend the string. e.g. “abcabbcc” | efg, from first ‘a’ to last ‘a’, we have characters ‘b’ and ‘c’, so we have to extend the string to cover all ‘b’s and ‘c’s. Our first valid substring extended from “abca” to “abcabbcc”.
  4. If any character in the covered range has a smallest first occurrence, then it’s an invalid substring. e.g. ab | “cbc”, from first ‘c’ to last ‘c’, we have ‘b’, but ‘b’ is not fully covered, thus “cbc” is an invalid substring.
  5. For the first valid substring, we append it to the ans array. “abbeefba” => ans = [“abbeefba”]
  6. If we find a shorter substring that is full covered by the previous valid substring, we replace that substring with the shorter one. e.g.
    “abbeefba” | ccc => ans = [“abbeefba”]
    “abbeefba” | ccc => ans = [“bbeefb”]
    “abbeefba” | ccc => ans = [“ee”]
  7. If the current substring does not overlap with previous one, append it to ans array.
    “abbeefba” | ccc => ans = [“ee”]
    “abbeefba” | ccc => ans = [“ee”, “f”]
    “abbeefbaccc” => ans = [“ee”, “f”, “ccc”]

1648. Sell Diminishing-Valued Colored Balls(销售价值减少的颜色球)

方法一： Greedy + Math

Sort the colors in the descending order.
Find a group of colors that have the same number of balls. We can sell them together to speed up the process until the # of balls equal to the next greatest color.
Time complexity: O(nlogn)
Space complexity: O(1)
cur: current number of balls
nxt: next number of balls
colors: # of colors have the same # of balls
((cur + nxt + 1) * (cur - nxt)/2) * colors
not enough orders to sell?
t = orders / colors ; r = orders / colors
nxt’ = cur - t
((cur + nxt’ + 1) * (cur - nxt’)/2) * colors + nxt’ * r

1838. Frequency of the Most Frequent Element(最高频元素的频数)

方法一：Sliding Window

Maintain a window [ij] such that it takes <= k ops to make all elements equal to nums[j]/target
target_sum = window.size() * nums[j]
actual_sum = sums(ij)
A window is valid if : target_sum - actual_sum <= k
Time complexity : O(nlogn) sorting + O(n) sliding window
space complexity : O(1)
