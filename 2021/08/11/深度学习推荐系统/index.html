6.3.1 Parameter Server的分布式训练原理

并行进行梯度下降计算，完成抹茶参数的更新直至最终收敛
数据并行训练产生局部梯度，再汇总梯度更新参数权重
server节点：保存模型参数、接收worker节点计算出的局部梯度、汇总计算全局梯度，并更新模型参数训练
worker节点：保存部分训练数据，从server节点拉取最新的模型参数，根据训练数据计算局部梯度，上传给server节点
Parameter Server的物理架构
服务节点组(server group):包含多个服务器节点(server node)，每个服务器节点负责维护一部分参数，服务器管理中(server manager)负责维护和分配server资源
工作节点组(worker group)：每个工作节点组对应一个Application(即一个模型训练任务)，工作节点组之间，以及工作节点组内部的任务节点之间并不通信，任务节点只与server通信
资源管理中心(resource manager)
Parameter Server的并行序列流程示意图
(1)push操作：worker节点利用本节点上的训练数据，计算好局部梯度，上传给server节点
(2)pull操作：为了进行下一轮的梯度计算，worker节点从server节点拉取最新的模型参数到本地

6.3.2 一致性与并行效率之间的取舍

异步非阻断式

6.3 推荐模型离线训练之Parameter Server

6.3.1 Parameter Server的分布式训练原理

并行进行梯度下降计算，完成抹茶参数的更新直至最终收敛
数据并行训练产生局部梯度，再汇总梯度更新参数权重
server节点：保存模型参数、接收worker节点计算出的局部梯度、汇总计算全局梯度，并更新模型参数训练
worker节点：保存部分训练数据，从server节点拉取最新的模型参数，根据训练数据计算局部梯度，上传给server节点
Parameter Server的物理架构
服务节点组(server group):包含多个服务器节点(server node)，每个服务器节点负责维护一部分参数，服务器管理中(server manager)负责维护和分配server资源
工作节点组(worker group)：每个工作节点组对应一个Application(即一个模型训练任务)，工作节点组之间，以及工作节点组内部的任务节点之间并不通信，任务节点只与server通信
资源管理中心(resource manager)
Parameter Server的并行序列流程示意图
(1)push操作：worker节点利用本节点上的训练数据，计算好局部梯度，上传给server节点
(2)pull操作：为了进行下一轮的梯度计算，worker节点从server节点拉取最新的模型参数到本地

7.1 离散评估方法与基本评估指标

离线环境下的评估：没有工程风险，无需浪费百鬼的线上资源，测试时间短，同时进行多组并行测试，能够利用丰富的线下计算资源等诸多优点

7.1.1 离线评估的主要方法

  1. Holdout检验
    训练集 + 验证集
    缺点：在验证集上计算出来的评估指标与训练集和验证集的划分有直接关系，结果存在随机性
  2. 交叉检验
    k-fold交叉验证：所有k词的评估指标的平均值作为最终的评估指标
    留一验证：n次验证，时间开销大
  3. 自助法
    基于自助采样法的检验方法：对于总数为N的样本集合，进行n次有放回的随机抽样，得到大小为N的训练集。在n词采样过程中，有的样本会被重复采样，有的样本没有被抽出过，讲这些没有被抽出的样本作为验证集进行模型检验

    7.1.2 离线评估的指标

  4. 准确率
    具有较强的可解释性，当类别分布不均衡是，占比大的类别称为影响准确率的最主要因素
  5. 精确率与召回率
    Precision: 分类正确的正确性个数占分类器判定为正样本的样本个数的比例
    Recall：分类正确的正样本个数占真正的正样本个数的比例
    TopN排序结果的精确率(Precision@)和召回率(Recall@N)
    F1-score：调和平均值
  6. 均方根误差
    均方根误差(Root Mean Square Error, RMSS)：衡量回归模型的好坏
    偏离程度非常大的离群点
    平均绝对百分比误差(Mean Absolute Percent Error, MAPE)
  7. 对数损失函数
    LogLoss：逻辑回归的损失函数
    适于观察模型收敛情况的评估指标

    7.2 直接评估推荐序列的离线指标

    7.2.1 P-R曲线

    横轴召回率
    纵轴精确率
    曲线上的点代表在某一阈值下，模型将大于该阈值的结果判定为正样本，将小于该阈值的结果判定为负样本时，排序结果对应的召回率和精确率
    曲线下面积(Area Under Curve, AUC)
    ###7.2.2 ROC曲线
    the Receiver Operating Characteristic曲线
    受试者工作特征曲线
    横坐标：False Positive Rate(假阳性率)
    纵坐标：True Positive Rate(真阳性率)

    7.2.3 平均精度均值

    mean Average Precision, mAP
    平均精度的计算只取正样本处的precision进行平均
    对所有用户的平均精度进行平均

    7.2.4 合理选择评估指标

    归一化折扣累计收益(Normalized Discounted Cumulti, NDCG)/覆盖率(coverage)/多样性(diversity)

    7.3 更接近线上环境的离线评估方法-Replay

    7.3.1 模型评估的逻辑闭环

    模型评估各环节的逻辑关系 离线评估的重点是让离线评估的结果能够尽量接近线上结果

7.3.2 动态离线评估方法

静态的：模型不会随着评估的进行而更新
增量学习更新时间点前的测试样本，更新后继续进行后续的评估

7.4 A/B测试与线上评估指标

验证新模块、新功能、新产品是否有效的主要测试方法

7.4.1 什么是A/B测试

分流测试 分桶测试 随机实验 实验组 对照组 控制变量
(1)离线评估无法完全消除数据有偏(data bias)现象的影响
(2)离线评估无法还原线上的工程环境
线上环境的延迟、数据丢失、标签数据缺失
(3)线上系统的某些商业指标在离线评估中无法计算
ROC曲线 P-R曲线
用户点击率 留存时长 PV访问量

7.4.2 A/B测试的分桶原则

样本的独立性和采样方式的无偏性：同一个用户在测试的全程只能被分到同一个桶中，在分桶过程中所有的用户ID应是一个随机数，这样才能保证桶中的样本是无偏的
实验流量分层和分流的机制
宝贵的线上流量的高可用性
(1)层与层之间的流量”正交”
层与层之间的独立实验的流量是正交的，即实验中每组的流量穿越该层后，都会被再次随机打散，且均匀地分布在下层实验的每个实验组中
层与层之间流量"正交"示例
(2)同层之间的流量”互斥”：如果同层之间进行多组A/B测试，那么不同测试之间的流量是不重叠的，即”互斥”的；一组A/B测试中的实验组和对照组的流量是不重叠的，是”互斥”的

7.4.3 线上A/B测试的评估指标

与线上业务的核心指标保持一致
离线评估不具备直接计算业务核心指标的条件，选择偏向于技术评估的模型相关指标
公司层面，更关心能够驱动业务发展的核心指标

7.5 快速线上评估测试方法-Interleaving

线上A/B测试占用宝贵的线上流量资源，对用户体验造成损害
Interleaving方法被当做线上A/B测试的预选阶段进行候选算法的快速筛选，从大量初始想法中筛选出少量”优秀”的推荐算法
使用Interleaving进行快速线上测试

7.5.1 传统A/B测试存在的统计学问题

效率问题 + 统计学上的显著性差异问题
不区分A/B分组，而把不同的被测对象同时提供给受试者，最终根据受试者喜好得出评估结果的方法就是Interleaving方法

7.5.3 Interleaving方法与传统A/B测试的灵敏度比较

灵敏度实验：验证Interleaving方法相比传统A/B测试，需要多少样本就能够验证出算法A和算法B的优劣

7.5.5 Interleaving方法的优点与缺点

(2)Interleaving方法只是对”用户对算法推荐结果偏好程度”的相对测量，不能得出一个算法真实的表现
