脚本阻塞问题实际有两种解决方案 —— async 和 defer
具有async属性的脚本将在下载完成后立即执行。这会阻止页面并且不保证任何特定的执行顺序。
具有defer属性的脚本将按照它们所在的顺序加载，并且只有在所有内容完成加载后才会执行。
如果脚本无需等待页面解析，且无依赖独立运行，那么应使用 async。
如果脚本需要等待页面解析，且依赖于其它脚本，调用这些脚本时应使用 defer，将关联的脚本按所需顺序置于 HTML 中。

Promise是由异步函数返回的对象，它表示操作的当前状态。在将promise返回给调用者时，操作通常还没有完成，但promise对象提供了处理操作最终成功或失败的方法。
promise的优雅特性是：then()本身返回一个promise，它将用传递给它的函数的结果来完成。
pending: the promise has been created, and the asynchronous function it’s associated with has not succeeded or failed yet. This is the state your promise is in when it’s returned from a call to fetch(), and the request is still being made.
fulfilled: the asynchronous function has succeeded. When a promise is fulfilled, its then() handler is called.
rejected: the asynchronous function has failed. When a promise is rejected, its catch() handler is called.

待处理(pending)：promise 已创建，其关联的异步函数尚未成功或失败。 这是您的 promise 从调用 fetch() 返回时的状态，并且请求仍在进行中。
A pending promise can either be fulfilled with a value or rejected with a reason (error).

A promise is resolved if it is settled, or if it has been “locked in” to follow the state of another promise.
