解析命令行选项、获取有用的系统配置数据

13.1 通过重定向/管道/文件接受输入

将命令行的输出通过管道传递给该脚本、重定向文件到该脚本，或在命令行中传递一个文件名或文件名列表给该脚本。
fileinput 模块

fileinput.input()创建并返回一个FileInput类的实例。

13.2 终止程序并给出错误信息

raise SystemExit(‘It failed!’)
将消息在sys.stderr中打印，然后程序以状态码1退出。

13.3 解析命令行选项

argparse 模块可被用来解析命令行选项。

手动处理sys.argv或者使用getopt模块；使用optparse库解析选项

13.4 运行时弹出密码输入提示

Python 的 getpass 模块：弹出密码输入提示，并且不会在用户终端回显密码

13.5 获取终端的大小

使用os.get_terminal_size()函数

读取环境变量；执行底层的ioctl()函数

13.6 执行外部命令并获取它的输出

使用subprocess.check_output()函数

使用shlex.quote()函数来将参数正确的用双引用引起来。
subprocess模块对于依赖TTY的外部命令不合适用。

13.7 复制或者移动文件和目录

shutil模块有很多便捷的函数可以复制文件和目录。

13.8 创建和解压归档文件

shutil模块拥有两个函数——make_archive()和unpack_archive()

13.9 通过文件名查找文件

os.walk()方法遍历目录树，每次进入一个目录，它会返回一个三元组，包含相对于查找目录的相对路径，一个该目录下的目录名列表，以及那个目录下面的文件名列表。
os.path.abspath()
os.path.normpath()
os.path.getmtime(fullpath)

13.10 读取配置文件

configparser 模块被用来读取普通.ini 格式的配置文件

配置文件作为一个整体读取，变量在使用它的变量之前或之后定义都是可以

13.11 给简单脚本增加日志功能

打印日志最简单方式是使用 logging 模块
critical(), error(), warning(), info(), debug()
basicConfig()的level参数是一个过滤器。所有级别低于此级别的日志消息都会被忽略掉。

13.13 实现一个计时器

time.perf_counter; time.process_time(); time.time() 或 time. clock()

13.14 限制内存和 CPU 的使用量

resource模块对在Unix系统上面运行的程序设置内存或CPU的使用限制

设置子进程数量、打开文件数以及类似系统资源的限制

13.15 启动一个 WEB 浏览器

webbrowser模块能被用来启动一个浏览器，并且与平台无关。
