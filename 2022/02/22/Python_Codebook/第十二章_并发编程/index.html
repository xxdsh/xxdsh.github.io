12.1 启动与停止线程

threading库可以在单独的线程中执行任何的在Python中可以调用的对象。
可以查询一个线程对象的状态，看它是否还在执行

12.2 判断线程是否已经启动

线程的一个关键特性是每个线程都是独立运行且状态不可预测
Event对象包含一个可由线程设置的信号标志，它允许线程等待某些事件的发生。

12.3 线程间通信

创建一个被多个线程共享的Queue对象，线程通过使用put()和get()操作来向队列中添加或者删除元素

12.4 给关键部分加锁

Lock对象和with语句块一起使用可以保证互斥执行
with相对于显示调用的缺点：程序员可能会忘记调用release()方法或者程序在获得锁之后产生异常

12.5 防止死锁的加锁机制

在进程获取锁的时候会严格按照对象id升序排列获取

12.6 保存线程的状态信息

使用thread.local()创建一个本地线程存储对象。对这个对象的属性的保存和读取操作都只会对执行线程可见，而其他线程并不可见。
每个threading.local()实例为每个线程维护着一个单独的实例字典。所有普通实例操作比如获取、修改和删除值仅仅操作这个字典。

12.7 创建一个线程池

concurrent.futures函数库有一个ThreadPoolExecutor类
全局解释器锁 GIL
当创建一个线程时，操作系统会预留一个虚拟内存区域来放置线程的执行栈(通常是8MB大小)
threading.stack_size(65536)
线程栈大小必须至少为32768字节

12.8 简单的并行编程

利用多核 CPU 的优势来运行
concurrent.futures库提供了一个ProcessPoolExecutor类，可被用来在一个单独的Python解释器中执行计算密集型函数。
一个 ProcessPoolExecutor 创建 N 个独立的 Python 解释器，N 是系统上面可用 CPU 的个数。

12.9 Python 的全局锁问题

Global Interpreter Lock (GIL)
解释器被一个全局解释器锁保护着，它确保任何时候都只有一个 Python 线程执行。
如果完全工作于Python环境中，可以使用multiprocessing模块来创建一个进程池，并像协同处理器一样的使用它。
使用 C 扩展编程技术

12.10 定义一个 Actor 任务

actor模式是一种最古老的也是最简单的并行和分布式计算解决方案。
一个actor就是一个并发执行的任务，只是简单的执行发送给它的消息任务。响应这些消息时，它可能还会给其他actor发送更进一步的消息。actor之间的通信是单向和异步的。

12.11 实现消息发布/订阅模型

要实现发布/订阅的消息通信模式，通常要引入一个单独的“交换机”或“网关”对象作为所有消息的中介。
