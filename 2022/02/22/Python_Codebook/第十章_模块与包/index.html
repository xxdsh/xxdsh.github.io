10.3 使用相对路径名导入包中子模块

from . import grok
from ..B import bar
相对导入只适用于在合适的包中的模块。尤其是在顶层的脚本的简单模块中，它们将不起作用。如果包的部分被作为脚本直接执行，那它们将不起作用
python3 mypackage/A/spam.py # Relative imports fail
python3 -m mypackage.A.spam # Relative imports work

10.4 将模块分割成多个文件

程序模块可以通过变成包来分割成多个独立的文件。
延迟加载的主要缺点是继承和类型检查可能会中断。

10.5 利用命名空间导入目录分散的代码

定义一个顶级Python包，作为一个大集合分开维护子包的命名空间。
包命名空间是一种特殊的封装设计，为合并不同的目录的代码到一个共同的命名空间。对
sys.path.extend([‘foo-package’, ‘bar-package’])
包命名空间的关键是确保顶级目录中没有 __init__.py 文件来作为共同的命名空间。
解释器创建了一个由所有包含匹配包名的目录组成的列表。
一个包是否被作为一个包命名空间的主要方法是检查其__file__ 属性。

10.6 重新加载模块

使用imp.reload()来重新加载先前加载的模块。
reload()擦除了模块底层字典的内容，并通过重新执行模块的源代码来刷新它。

10.9 将文件夹加入到 sys.path

使用 PYTHONPATH 环境变量来添加
bash % env PYTHONPATH=/some/dir:/other/dir python3

10.10 通过字符串名导入模块

10.11 通过钩子远程加载模块

加载远程模块的第一种方法是创建一个显示的加载函数来完成它。
