15.1 使用 ctypes 访问 C 代码

使用 Python 标准库中的 ctypes 模块
确保你要访问的C代码已经被编译到和Python解释器兼容(同样的架构、字大小、编译器等)的某个共享库中了。

15.2 简单的 C 扩展模块

PyObject 是一个能表示任何 Python 对象的 C 数据类型。
在一个高级层面，一个扩展函数就是一个接受一个 Python 对象(在PyObject *args中)元组并返回一个新Python对象的C函数。
PyArg_ParseTuple() 函数被用来将 Python 中的值转换成 C 中对应表示。
Py_BuildValue() 函数被用来根据 C 数据类型创建 Python 对象。

15.6 从 C 语言中调用 Python 代码

PyGILState_Ensure() 可以确保调用线程独占 Python 解释器。
调用成功后，PyGILState_Release()被用来将解释器恢复到原始状态。

15.7 从 C 扩展中释放全局锁

只有当你确保没有Python C API函数在C中执行的时候才能安全的释放 GIL。
Py_BEGIN_ALLOW_THREADS
// Threaded C code. Must not use Python API functions
Py_END_ALLOW_THREADS

15.8 C 和 Python 中的线程混用

PyGILState_STATE state = PyGILState_Ensure();
PyGILState_Release(state);
调用PyGILState_Ensure()并不会立刻抢占或中断解释器。
