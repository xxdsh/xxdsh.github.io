5.1 读写文本数据

使用带有rt模式的open()函数读取文本文件。
系统编码 sys.getdefaultencoding()
ascii, latin-1, utf-8 和 utf-16
当读取一个未知编码的文本时使用 latin-1 编码永远不会产生解码错误。
参数 newline=’’ 参数 encoding=’ascii’ 参数 errors=’replace’

5.2 打印输出至文件中

文件必须是以文本模式打开。
print(‘Hello World!’, file=f)

5.3 使用其他分隔符或行终止符打印

5.4 读写字节数据

使用模式为rb或wb的open()函数来读取或写入二进制数据。
索引和迭代动作返回的是字节的值而不是字节字符串。
从二进制模式的文件中读取或写入文本数据，必须确保要进行解码和编码操作。
数组和 C 结构体类型能直接被写入
”缓冲接口” 直接暴露其底层的内存缓冲区给能处理它的操作
文件对象的readinto()方法直接读取二进制数据到其底层的内存中去

5.5 文件不存在才能写入

可以在 open() 函数中使用 x 模式来代替 w 模式

5.6 字符串的 I/O 操作

使用io.StringIO()和io.BytesIO()类来创建类文件对象操作字符串数据。
StringIO和BytesIO实例并没有正确的整数类型的文件描述符。

5.7 读写压缩文件

gzip.open(‘somefile.gz’, ‘rt’)
bz2.open(‘somefile.bz2’, ‘rt’)
写入压缩数据时，可以使用compresslevel这个可选的关键字参数来指定一个压缩级别。
默认的等级是9，也是最高的压缩等级。等级越低性能越好，但是数据压缩程度也越低。

5.10 内存映射的二进制文件

内存映射一个二进制文件到一个可变字节数组中
mmap模块
mmap()返回的mmap对象同样也可以作为一个上下文管理器来使用
mmap.ACCESS_COPY mmap.ACCESS_READ
内存映射一个文件并不会导致整个文件被读取到内存中。

5.21 序列化 Python 对象

pickle 模块。 pickle 是一种 Python 特有的自描述的数据编码。
通过自描述，被序列化后的数据包含每个对象开始和结束以及它的类型信息。
